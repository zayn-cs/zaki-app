CONTEXTE DU PROJET :
Il s’agit d’un projet de fin d’études : conception et développement d’une application d’archivage performant des projets d’infrastructures avec recherche multicritère avancée.
La hiérarchie métier principale du système est :
Projet → Lot → Phase → Document → Version

Le système doit permettre :
- la gestion des départements
- la gestion des projets
- la gestion des lots
- la gestion des phases
- la gestion des documents
- la gestion des versions des documents
- l’archivage des documents
- la recherche multicritère performante
- la traçabilité / historique des actions
- le contrôle d’accès selon les rôles

1. Authentification
- page login

- gestion de session
- redirection selon le rôle

2. Tableau de bord
- dashboard adapté à chaque rôle
- statistiques simples :
  - nombre de projets
  - nombre de lots
  - nombre de phases
  - nombre de documents
  - nombre de documents archivés
  - éventuellement activité récente

3. Gestion des entités métier
- CRUD sur :
  - départements
  - projets
  - lots
  - phases

  - utilisateurs

4. Gestion des documents

6. Archivage

- changer son statut en “archivé”

- statut

Recherche sur les documents selon :
- projet
- lot
- phase
- type
- date
- version

Le moteur de recherche doit :
- combiner plusieurs critères

8. Historique / traçabilité

CONTRAINTE IMPORTANTE SUR LES DIAGRAMMES/FONCTIONNEMENT :
Je dispose déjà d’une analyse UML et je veux que tu respectes l’idée fonctionnelle suivante :

Séquence upload :
- utilisateur sélectionne un fichier
- frontend envoie la requête upload au backend
- backend vérifie l’authentification
- backend valide les données
- backend enregistre le fichier dans le stockage
- backend enregistre le document et sa version dans la base
- retour succès au frontend

Séquence recherche :
- utilisateur saisit les critères
- frontend envoie une requête de recherche
- backend construit une requête multicritère dynamique
- backend exécute la recherche dans la base
- backend trie les résultats
- frontend affiche la liste des documents/projets

Séquence archivage :
- utilisateur choisit un document à archiver
- frontend envoie la requête d’archivage
- backend vérifie les droits d’accès
- backend met à jour le statut du document en “archivé”
- backend enregistre l’historique de l’action
- frontend affiche une confirmation
